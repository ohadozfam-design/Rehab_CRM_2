# Step 2: Authentication, Layout & Theme Engine
Goal: Build the multi-role Login view, main App Layout shell, and time-based Theme Engine.

## Task Details
1. **Login Page (`/login`):**
   - Exact replica of the login mockup. Card layout centered on a clean gradient screen.
   - Display clickable "Demo Account" badges that automatically fill fields and log in.
2. **App Shell & Layout:**
   - Top navigation bar featuring the logo, title ("Rehab CRM"), user profile indicator with role badge, and actions toolbar (Theme toggle, notification bell with unread indicator badge, settings, users).
3. **Theme Switcher Component:**
   - Supports Light, Dark, and Auto.
   - Auto mode dynamically checks time: Dark from 18:00 to 07:00, Light otherwise.
4. **Role Gating System:**
   - Detect active user role. Filter visible projects automatically on entry.
   - Set up standard navigation layout where views adapt or throw "Access Denied" blocks if a user does not have specific responsibilities.

## Definition of Done
- Complete login flow working seamlessly. Theme switcher updates HTML data attributes accurately matching CSS tokens.
- Progress documented in `zeadim/02_auth_and_shell.md`, pushed to GitHub. Stop and wait for user approval.
