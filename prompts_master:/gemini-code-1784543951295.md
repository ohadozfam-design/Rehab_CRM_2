# Step 6: Photos Gallery, Slack-style Communication & Background Notification Engine
Goal: Program the media asset sections, localized message timelines, background automation loops, and daily reports.

## Task Details
1. **Photos & Media Tab:**
   - Segmented layout organizing photos by project phase. Includes dedicated inputs to link shared Google Drive folders.
   - Media cards show preview thumbnails with absolute delete overrides.
2. **Updates Feed Tab:**
   - Private communication logging stream. Posts display author name, role subtitle, phase tags, message contents, and creation timestamps.
3. **Background Notification Automation Loop:**
   - Initialize a unified background observer engine simulating an active cron layer (running via `setInterval` inside app context).
   - **Rule 1:** Check assigned SOW items. If marked complete (100%) but contains ZERO media proof attachments, fire an automated `item-proof-needed` warning notice to the worker twice daily.
   - **Rule 2:** Check transactional logs. If a payment requires a lien waiver but lacks an upload url, fire a `lien-waiver-needed` task reminder daily.
   - **Rule 3:** Monitor overruns. Trigger system-wide critical dashboard alerts when budgets blow out.
4. **Morning Snapshot Dialog:**
   - A welcoming pop-up modal triggering automatically once a day after 08:00 AM. Summarizes current holdings, active expenditures, weekly progress, and flags critical items at risk.

## Definition of Done
- Media galleries load items dynamically. The notification scheduler actively generates and clears logs based on context shifts.
- Progress documented in `zeadim/06_automation_final.md`, pushed to GitHub. Stop and wait for user approval.
