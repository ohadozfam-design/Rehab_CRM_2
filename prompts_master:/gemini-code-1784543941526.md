# Step 1: Architecture & Global State Setup
Goal: Initialize the React application and build the full Zustand state architecture with LocalStorage persistence.

## Task Details
1. Set up a standard React + Vite + Tailwind project.
2. Create the `zeadim/` folder to track steps.
3. Implement the 6 core Zustand stores exactly as defined in the data model spec:
   - `useAuthStore` (key: `rehab-crm-auth`): Manages `users[]`, `currentUserId`. Include pre-loaded demo accounts:
     * `admin` / `admin` (Role: admin)
     * `manager` / `manager` (Role: manager - assigned: Pursell, Clay)
     * `doolin` / `doolin` (Role: contractor - assigned: Pursell)
     * `eran` / `eran` (Role: contractor - assigned: Clay)
     * `viewer` / `viewer` (Role: viewer)
   - `useRenovationStore` (key: `rehab-crm-storage`): Manages `renovations[]` matching the full schema (id, name, address, phases, sowItems, financialEntries, receipts, updates).
   - `useNotificationStore` (key: `rehab-crm-notifications`): Manages `notifications[]`.
   - `useContactsStore` (key: `rehab-crm-contacts`): Address book for automated contact picker.
   - `useSettingsStore` (key: `rehab-crm-settings`): User preferences (Morning snapshot timing).
   - `useThemeStore` (key: `rehab-crm-theme`): Light/dark/auto tracking.

## Definition of Done
- App builds successfully with zero TypeScript or compilation errors.
- Global stores initialized with mock data matching the spec (Pursell Ave, Clay St, Pine Ridge Ct).
- Progress documented in `zeadim/01_state_architecture.md`, pushed to GitHub. Stop and wait for user approval.
