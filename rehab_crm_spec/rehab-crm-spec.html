Lien waiver enforcement

Any expense > $600 in category labor or material auto-flags lienWaiverRequired: true. Notification engine sends 1 daily reminder to the assigned contractor until lienWaiverUrl is uploaded. Badge cycles: ⚠ WAIVER NEEDED → ✓ WAIVER ON FILE

15 Store Architecture

Six independent Zustand stores. All are persisted to localStorage.

StorePersistent keyStatePurpose
useAuthStorerehab-crm-authusers[], currentUserIdUsers, login/logout, CRUD
useRenovationStorerehab-crm-storagerenovations[]All project data. All mutations flow through here.
useNotificationStorerehab-crm-notificationsnotifications[]User-scoped alerts; engine runs every 60s
useContactsStorerehab-crm-contactscontacts[]Address book of contractors + managers
useSettingsStorerehab-crm-settingsbyUserId per-user preferencesMorning snapshot toggle, snapshot time
useThemeStorerehab-crm-thememode: 'light'|'dark'|'auto'Theme toggle, auto by time-of-day

Notification engine flow

// Runs once per minute via useEffect(setInterval)
for each renovation:
for each SOWItem with assignedUserId:
if completed && media.length > 0 → resolve existing reminder
else if hoursSince(lastFire) >= 12 → fireReminder(item-proof-needed)
for each FinancialEntry with lienWaiverRequired:
if lienWaiverUrl → resolve
else if hoursSince(lastFire) >= 24 → fireReminder(lien-waiver-needed)
for each overrun (budget or schedule):
for each admin/manager on project:
if condition true → fireReminder(critical)
else → resolve
